三个步骤, 开启自动化之旅

从繁琐的手动工作流中解脱，只需三步即可完成

1. 导入 SKU

通过文本粘贴或 TXT 文件一键导入需要申诉的商品清单，系统自动解析 ID。

2. 匹配证据图片

在本地图片库中上传授权凭证，应用将根据侵权类型自动关联最佳申诉材料。

3. 插件自动填报

启动任务后，浏览器插件将自动接管 Ozon 后台页面，执行高频模拟提交。

Step 01

导入订单

用户粘贴或导入 SKU 列表，任务写入 pending 队列。

Step 02

拉取待处理任务

插件请求 unprocessed，原子化把 pending 切换为 processing。

Step 03

插件自动填报

输入 SKU、上传凭证图片并提交，持续轮询页面状态。

Step 04

结果判定

先读取页面结果文案，识别成功或失败分支，避免误判。

Step 05

状态回写 API

调用 complaint/:sku/status，将结果与 chatId 备注同步到宿主。

Step 06

广播与下一轮

数据库更新广播到 UI，插件继续拉取下一条任务。

常见问题

为您解答关于 OzonAssist 的一切疑问

使用本软件会不会导致封号？

软件完全模拟真实的人类操作流程，通过本地 API 与插件进行交互，不涉及篡改 Ozon 后台核心逻辑，安全性极高。我们已服务 1200+ 卖家，目前保持零风险记录。

是否支持 macOS 平台？

是的。OzonAssist 基于 Electron 开发，完美适配 macOS (包括 M1/M2/M3 及 Intel 芯片) 和 Windows 10/11 系统。

图片资料是存储在服务器上的吗？

不是。出于隐私考虑，所有的图片和 SKU 数据均存储在您本地的 SQLite 数据库中。除非您在使用过程中通过浏览器提交，否则任何数据都不会上传至云端。
